>

> - audio eval: `eval_runs/20260530-120221-audio-test/` (5 L2、$0.0118、STT 動作確認済)

## 撮影の全体方針

1. **本物の Teams 会議で撮る**: 2 台の PC が Teams 会議に参加。Helmsman bot を 3 番目の参加者として派遣
2. **2 人の擬似参加者を音声で再現**: スマホ A から `yamada.wav` (Daichi 男声)、スマホ B から `sato.wav` (Mayu 女声) を **同時再生**。スマホは各 PC のマイクに向ける
3. **音声は被らない設計**: 32 発言を yamada 19 + sato 13 で交互配置、`offset_sec + duration_sec` ≤ 次の `offset_sec` を厳守 (検証済 0 件)
4. **UI は本物**: 撮影用 PC とは別に、もう 1 台で Helmsman ダッシュボードを開き画面録画
5. **L3 音声介入**: 本物の Helmsman bot が Azure TTS で「残り 10 分です…」を発話 (会議に直接音声で流れる)
6. **ナレーション**: あとから iMovie で重ねる (撮影時は無音 OK)

### 画面録画
- **macOS Screen Recording**: ⌘+Shift+5 → "選択した部分を収録" → ブラウザ全体を指定
- **音声**: 「マイク (Built-in)」を選択しておく (Helmsman bot の L3 音声が PC スピーカーから流れるのを拾う)
- **解像度**: 1920×1080 出力、Retina ディスプレイなら自動

### 音声同期
- 2 台のスマホは **タイマーで 3-2-1-スタート** の合図で同時再生
- 1 秒以内の誤差なら聞いて分からない
- スマホアプリ: 「タイマー」標準アプリで OK

### TTS 音声生成 (撮影前 1 回だけ)
- `uv run python scripts/gen_demo_audio.py` で 3 ファイル生成済み
- yamada.wav → AirDrop でスマホ A に転送
- sato.wav → AirDrop でスマホ B に転送
- l3_intervention.wav は **本物の Helmsman bot が生成するので使わない** (Scene 7 で会議中に自動発火)

### 編集 (iMovie)
- カット編集で間延びを削る (3:53 → 3:00)
- ズーム/ハイライト効果でクローズアップ感を演出
- BGM (royalty-free) を低音量で

## 失敗時のフォールバック

| シーン | 失敗時の代替 |
|---|---|
| Bot 派遣が失敗 | `scripts/eval_offline.py --transcript ...` の出力 (`eval_runs/20260530-114702-demo-take4/`) を画面表示 → ナレーションで補完 |
| Scene 7 で L3 が発火しない | 別途生成済の `eval_runs/demo-audio/l3_intervention.wav` を該当タイミングで再生 + InterventionFeed に手動でカード追加 |
| Scene 8 でレポート生成が遅い | 事前生成した markdown を表示画面に切り替え |
| 2 人のスマホ同期がずれる | 1 つの音源を 1 台の PC で再生し、別 PC では普通に黙って Teams 参加 (1 人会議扱い) |
